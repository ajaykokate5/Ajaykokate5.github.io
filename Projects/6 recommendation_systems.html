Data Description:  ¶

Amazon Reviews data The repository has several  datasets. For this case study, we are using the Electronics  dataset. ¶

Domain:  ¶

E-commerce¶

Context: ¶

Online E-commerce websites like Amazon, Flipkart uses  different recommendation models to provide different  suggestions to different users. Amazon currently uses  item-to-item collaborative filtering, which scales to massive  data sets and produces high-quality recommendations in  real-time. ¶

Attribute Information: ¶

  • userId : Every user identified with a unique id
  • productId : Every product identified with a unique id
  • Rating : Rating of the corresponding product by the corresponding user
  • timestamp : Time of the rating ( ignore this column for this exercise)

Learning Outcomes:  ¶

  • Exploratory Data Analysis
  • Creating a Recommendation system using real data
  • Collaborative filtering

Objective ¶

Build a recommendation system to recommend products to  customers based on the their previous ratings for other  products. ¶

Function to recommend top 5 products¶

In [28]:
def recommend5(predictions, topN = 5): #function definition
     
    recommendations = defaultdict(list) #create a dicttionary of list type
    for uid, iid, true_r, est, _ in predictions: #for loop
        recommendations[uid].append((iid, est))#append id
     
    for uid, user_ratings in recommendations.items():#for loop
        user_ratings.sort(key = lambda x: x[1], reverse = True)#sort ratings
        recommendations[uid] = user_ratings[:topN]#select top 5
     
    return recommendations #return

def read_item_names(): #function definition
    file_name = ('ratings_Electronics.csv')#store filename
    names = {}#empty dictionary
    with io.open(file_name, 'r', encoding='ISO-8859-1') as f: #open file
        for line in f: #read data linewise
            line = line.split(',')#split using ','
            names[line[1]] = line[1]#assign value
 
    return names #return

Conclusion:¶

  • Insights and comments are already added after each step. These are some additives in the documentation.

Comments on Dataset:¶

  • The dataset is large as it has around 7.9M rows.
  • A challenge is to reduce the dataset without information loss. It is subject to either domain expertise or more information about the data filters.
  • In this analysis, users who have given more than 50 ratings and products which have more than 50 ratings are kept. A filter about the users is metioned in the project document specification.
  • The 'timestamp' column has been excluded from this exercise as per the project document specification.
  • Dataset has 1 to 5 rating scale

Comments on Models:¶

  • Baselineonly and SVD produced lowest RMSE while in KNN based models KNNbaseline produced lowest RMSE.
  • As per theory, lowest RMSE is expected from models so for this dataset any algorithm from baselineonly, SVD and KNNbaseline can be chosen; But KNNbaseline should be preffered as it provides an easy way to configure models for user based or item based collaberative filtering.
  • Recommendations for each model differ as their Error values differ.
  • A difference is noticeable on the recommendations based on user/item based model.
  • SVD has higher time complexity than other models.

Miscellaneous comments:¶

  • The Execution of large dataset without applying filters was causing 'memory error' while executing.
  • The correlationplot,pairplot do not provide significant insights into this kind of dataset, also these plots have huge time complexity so they are not included in this analysis.
