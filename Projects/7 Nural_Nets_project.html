The Real Problem¶

Recognizing multi-digit numbers in photographs captured at street level is an important component of modern-day map making. A classic example of a corpus of such street level photographs is Googles Street View imagery comprised of hundreds of millions of geo-located 360 degree panoramic images. The ability to automatically transcribe an address number from a geo-located patch of pixels and associate the transcribed number with a known street address helps pinpoint, with a high degree of accuracy, the location of the building it represents. More broadly, recognizing numbers in photographs is a problem of interest to the optical character recognition community. While OCR on constrained domains like document processing is well studied, arbitrary multi-character text recognition in photographs is still highly challenging. This difficulty arises due to the wide variability in the visual appearance of text in the wild on account of a large range of fonts, colors, styles, orientations, and character arrangements. The recognition problem is further complicated by environmental factors such as lighting, shadows, specularities, and occlusions as well as by image acquisition factors such as resolution, motion, and focus blurs. In this project we will use dataset with images centred around a single digit (many of the images do contain some distractors at the sides). Although we are taking a sample of the data which is simpler, it is more complex than MNIST because of the distractors.

The Street View House Numbers (SVHN) Dataset¶

SVHN is a real-world image dataset for developing machine learning and object recognition algorithms with minimal requirement on data formatting but comes from a significantly harder, unsolved, real world problem (recognizing digits and numbers in natural scene images). SVHN is obtained from house numbers in Google Street View images.

Model Building¶

Create a Sequential model in Keras with input layer with the correct input shape, Hidden Layers, Output Layers and the activation functions¶

  • Following is a sequential model with 1 input layer, 1 output layer and 5 hidden layers.
  • The network is dense i.e. fully connected and each layer has different number of neurons.
  • BatchNormalization is added after each layer before activation.
  • LeakyReLU and softmax activations are used in the network.
  • Dropout is added randomely after some layers.
  • 'he_normal' kernel initializer is used.
In [8]:
def build_basic_model():
        
    model = Sequential() #sequential network
    model.add(Dense(1024, input_shape=(1024,),kernel_initializer='he_normal'))#dense layer eith 1024 neurons
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leakyrelu activation


    model.add(Dense(512,kernel_initializer='he_normal'))
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leakyrelu activation

    model.add(Dense(256, kernel_initializer='he_normal'))
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leakyrelu activation
    model.add(Dropout(0.3))

    model.add(Dense(128,kernel_initializer='he_normal'))
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.3))#leakyrelu activation


    model.add(Dense(64,kernel_initializer='he_normal'))
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.3))#leakyrelu activation
    model.add(Dropout(0.3))#dropout

    model.add(Dense(32,kernel_initializer='he_normal'))
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.3))#leakyrelu activation
    model.add(Dropout(0.3))#dropout    

    model.add(Dense(10, activation='softmax'))    #dense layer with softmax activation
        
    return model

def build_model_regulariztion(lam):
    model = Sequential() #sequential network
    model.add(Dense(1024, input_shape=(1024,),kernel_regularizer=l2(lam)))#dense layer with 1024 neurons
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.1))#dropout   


    model.add(Dense(512,kernel_initializer= 'he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.1))#dropout 


    model.add(Dense(256,kernel_initializer='he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.1))#dropout 


    model.add(Dense(128,kernel_initializer='he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.3))#dropout 

    model.add(Dense(64,kernel_initializer='he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.1))#dropout 
    
    model.add(Dense(32,kernel_initializer='he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.2))#dropout   
    

    model.add(Dense(10, activation='softmax')) #dense layer with softmax activation  
    return model

def build_model_regulariztion(lam):
    model = Sequential()#sequential network
    model.add(Dense(1024, input_shape=(1024,),kernel_initializer= 'he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.1))#dropout       


    model.add(Dense(512,kernel_initializer= 'he_normal',kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.3))#leaky relu
    model.add(Dropout(0.1))#dropout

    model.add(LeakyReLU(alpha=0.3))#leaky relu
    model.add(Dropout(0.1))#dropout

    model.add(BatchNormalization())#batch normalization
    model.add(LeakyReLU(alpha=0.1))#leaky relu
    model.add(Dropout(0.3))#dropout

def create_model(dropout_rate,optimizer,init_mode,activation,lam):
    model = Sequential()#sequential network
    
    model.add(Dense(1024, input_shape=(1024,),kernel_initializer= init_mode,activation=LeakyReLU(),kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(Dropout(dropout_rate))#dropout       


    model.add(Dense(512,kernel_initializer= init_mode,activation=activation,kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(Dropout(dropout_rate))#dropout    


    model.add(Dense(256,kernel_initializer=init_mode,activation=activation,kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(Dropout(dropout_rate))#dropout    


    model.add(Dense(128,kernel_initializer=init_mode,activation=activation,kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(Dropout(dropout_rate))#dropout    

    model.add(Dense(64,kernel_initializer=init_mode,activation=activation,kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(Dropout(dropout_rate))#dropout    
    
    model.add(Dense(32,kernel_initializer=init_mode,activation=activation,kernel_regularizer=l2(lam)))#dense layer
    model.add(BatchNormalization())#batch normalization
    model.add(Dropout(dropout_rate))#dropout      
    

    model.add(Dense(10, activation='softmax')) #dense layer    
    

    model.compile(loss='categorical_crossentropy', optimizer=optimizer,  metrics=['acc']) #compile model with accuracy metrics 
    return model #return model
  • Only LeakyReLU is kept in activation because it has outperformed all other activations in the trial runs before finalizing the project and to reduce the execution time now.

Drop based Learning rate scheduler¶

  • This scheduler will help in automatically adjusting learning rate using a drop function.
  • The function is designed to modify learning rate after 10 epochs.
  • Initial learning rate is 0.1 which will be reduced by 0.5 after 10 epochs.

initial_lrate = 0.1 #initial learning rate
drop = 0.5 #drop size
epochs_drop = 10.0 #drop after 10 epochs

def scheduler(epoch):    #define function
    lrate = initial_lrate * math.pow(drop, math.floor((1+epoch)/epochs_drop)) #drop function
    return lrate #return     

callback = tf.keras.callbacks.LearningRateScheduler(scheduler) #keras callback object

Comments¶

  • All insights, important observation have alredy been added below every step. These are some concluding comments.

Comments on Dataset¶

  • The dataset is of h5py type, which is herarchical file type.
  • All the train, test and validation data was already present in splits, so no train_test_val split is performed.
  • The Number of samples in train test and validation are unequal.
  • The networks are trained on 42000 samples, validated on 60000 samples and tested on 18000 samples.

Comments on Networks¶

  • At first, The model without any regularization performed well than model with large lambda(Regularization) value.
  • The number of hidden layers and neurons in each hidden layer are chosen at random, but for input and output layer the number of neurons depended on number of input and output classes.
  • The Loss of validation and training should be almost equal in the end, which is achieved in the last network, which produced around 88% accuracy.

Miscellaneous Comments¶

  • GridSearchCV lead to infinite loop while training the parameters, the jupyter kernel used to disconnect after 4-5 hours.
  • On google colab, GridSearchCV ran for almost 12 hours before being disconnected.
  • Hyperparameters could have been more tuned with gridsearchCV or hyperas packages with a lot of time and infrastructure to execute all possible grids with parameters such as number of hidden layers, number of neurons, activations, dropout, regularization, batch size, epochs, and optimizer.
  • Tuning a single parameter at a time did not produce optimum results after combining all hyperparameters in a network, the loss and accuracy exploded.
  • I tried talos,hyperas and gridsearchCV for optimization, but due to lack of infrastructure, the functions were not able to tune all parameters at ones.
  • RandomSearch Doesnot guarantee the best set of hyperparameters, so I had to use a default netowrk with tuned learning rate to produce higher results.
  • Only LeakyReLU is used in RandomizedSearchCV because of its performance in previous iterations and to save time while finalizing the project.
